    GSEA-LITE

    gsea-lite 是一个专为转录组数据分析设计的轻量级基因集富集分析 Python 工具库。本项目完整实现了两种生信核心富集分析算法：过表达分析 (ORA) 与 简化版基因集富集分析 (GSEA)，并集成了多重假设检验校正（BH-FDR）与高质量的科研数据可视化功能。

    1、 小组成员

    • 丁奕新：数据输入与过表达分析模块（io.py, ora.py）
    • 夏奕琳婷：多重检验与简化版基因集富集模块（stats.py, gsea.py）
    • 于润湉：图形绘制、包封装与代码测试（visualize.py, cli.py, __init__.py,tests/,demo.ipynb）

    2、核心功能

    1. 标准 GMT 文件解析：手工实现高容错率的 GMT 格式文件解析器，支持空字符清洗与非规范行剔除。
    2. 差异表达数据校验：基于 pandas 读取真实 GEO/TCGA 差异表达结果，具备严格的核心列名（Gene.symbol, logFC, P.Value）和空值过滤校验。
    3. 经典过表达分析 (ORA)：自定义全局背景基因库（Universe），利用 scipy.stats 超几何生存函数（Survival Function）精准计算每个通路的原始 $p$-value。
    4. Benjamini-Hochberg (BH) FDR 校正：原生算法实现多重假设检验校正，通过严谨的递增保护逻辑确保 FDR 随着原 $p$-value 递增而保持绝对单调性。
    5. 简化版 GSEA 富集分数计算：根据表达变化幅度（$\log_2\text{FC}$）对全量基因进行降序排列，利用随机行走算法实时捕捉基因集在列表顶端或底端的最大偏离峰值（ES）。
    6. 科研级图形可视化：自动筛选 Top 10 显著富集通路，输出符合学术期刊发表标准的富集柱状图（横轴为 $-\log_{10}(\text{FDR})$）与多维映射气泡图（映射 Gene Ratio, Count 和 FDR）。

    4、环境准备与安装

    在使用工具包之前，请确保您的环境中已安装 Python 3.8+，并安装以下第三方依赖：

    方法一： Conda 环境文件 (environment.yml)

    conda env create -f environment.yml

    environment.yml 核心依赖包含：

    pandas>=1.3.0
numpy>=1.20.0
scipy>=1.7.0
matplotlib>=3.4.0
seaborn>=0.11.0
pytest>=6.0.0

    方法二：Python 依赖列表 (requirements.txt)

    pip install -r requirements.txt

    方法三：开发者模式安装本项目

    如果希望使用命令行入口 gsea-lite，建议在项目根目录运行：

    pip install -e .

    如果还需要运行测试和 Notebook：

    pip install -e ".[dev]"

    5、使用指南 (CLI 命令行接口)

    本项目提供了易用的命令行入口 gsea-lite，用户可以通过终端命令一键运行分析。

    1. 运行过表达分析 (ORA)

    利用显著差异基因（默认阈值 $p < 0.05$ 且 $|\log_2\text{FC}| > 1$）和全局背景基因库进行超几何检验：

    gsea-lite ora --geo examples/example_data/GSE42568.top.table.tsv --gmt examples/example_data/h.all.v2026.1.Hs.symbols.gmt --outdir results
    • 输出成果：运行成功后，将在 results/ 目录下自动生成 ora_results_fdr.csv、ora_barplot.png、以及 ora_dotplot.png。

    2. 运行简化版基因集富集分析 (GSEA)

    基于全量表达谱基因的 $\log_2\text{FC}$ 降序排列列表，计算各通路的富集分数（ES）：

    gsea-lite gsea --geo examples/example_data/GSE42568.top.table.tsv --gmt examples/example_data/h.all.v2026.1.Hs.symbols.gmt --outdir results
    • 输出成果：将在 results/ 目录下生成包含各通路最大富集波峰/波谷分数的 gsea_results.csv。

    6、自动化单元测试

    为确保跨模块协作时的算法准确性及边界防御力，项目配备了完善的单元测试。您可以通过以下命令在根目录下运行测试：

    pytest tests/

    测试集涵盖了以下 5 大核心场景的自动化质检：

    1. GMT 边界解析测试：确保解析器能准确容错包含空行、尾部空格以及残缺基因的 GMT 行。
    2. DEG 列名强校验测试：当输入的 CSV/TSV 文件缺失 P.Value 等核心关键列时，程序必须能抛出格式异常。
    3. ORA 显著筛选测试：严格复核筛选器是否精准截取满足双重阈值的基因。
    4. BH-FDR 单调递增保护测试：验证校正后的 FDR 值在保证不小于原 $p$-value 的同时，随着原 $p$-value 升序必须呈现绝对的单调非递减属性。
    5. GSEA 随机行走路径测试：输入特定的模拟基因集，动态复推 ES 步进累加与扣除过程，验证最终捕捉的绝对峰值与理论数学推导完全一致。

    7、生物学背景知识库

    • Background Gene Universe（背景基因库）的影响：ORA 分析强烈依赖于“总体基因大小 $M$”的定义。若误将全基因组所有基因作为背景，而实际上芯片或测序只检测了部分表达基因，会导致分母偏大，从而使超几何检验的 $p$-value 假性显著。本项目严格提取测序结果中实际检测到的去重基因集作为背景宇宙。
    • 多重假设检验校正的必要性：当同时对几百个通路独立进行超几何检验时，发生假阳性（I类错误）的概率会随通路数量增加而剧烈膨胀。使用 BH-FDR 校正能有效平衡假阳性与假阴性，筛选出真正具有生物学意义的通路。
